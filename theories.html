Norman’s Model of Interaction

Norman describes interaction as a cycle between the user and the system.

  • Forming the goal
  • Forming the intention
  • Specifying the action
  • Executing the action
  • Perceiving the system state
  • Interpreting the system state
  • Evaluating the outcome

Reference: Dix et al., HCI, 3rd ed., Chapter 9, pp. 307–314

Gulf of Execution:
The gap between what the user wants to do and what actions the system allows or makes visible.

Example:
A user wants to print a document but cannot find the “Print” option because it is hidden in a confusing menu.

Gulf of Evaluation:
The gap between what the system does and how well the user can understand the result.

Example:
After clicking “Submit”, nothing seems to happen, so the user is unsure whether the form was sent.
