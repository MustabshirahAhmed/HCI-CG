1. Don Norman – Model of Interaction

Don Norman is a cognitive scientist who studied how people think and behave when using interactive systems. His model explains interaction as a cycle between the user and the system.

Norman’s model is mainly used to explain: why users get confused, why errors happen, and how design can reduce these problems.

The Seven Stages of Action

  1. Forming the goal
  2. Forming the intention
  3. Specifying the action
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the outcome

Gulf of Execution

The Gulf of Execution is the gap between what the user wants to do and what actions the system allows or makes visible.

Example: A user wants to turn Wi-Fi ON, but the icon is hidden in a confusing menu. The goal is clear, but the user doesn’t know how to do it.

Gulf of Evaluation

The Gulf of Evaluation is the gap between what the system does and how well the user understands it.

Example: The user clicks “Submit” on a form, but nothing changes on the screen. The system gives no feedback, so the user doesn’t know if it worked.

Reference:

Dix et al., Human–Computer Interaction, 3rd ed., Pearson. Chapter 9, pp. 307–314 (Norman’s Model, Gulfs of Execution & Evaluation)

2. GOMS Model (Goals, Operators, Methods, Selection Rules)

The GOMS model was proposed by Card, Moran, and Newell. It explains how an expert user performs a task after learning is complete. It is a task-analysis model that describes structured task performance.

  • Goals: What the user wants to achieve
  • Operators: Basic actions (click, type, move mouse)
  • Methods: Step-by-step procedures to achieve a goal
  • Selection Rules: Rules for choosing between methods

Example: Goal: Withdraw cash Method: Insert card → Enter PIN → Select Withdraw → Enter amount Operators: Insert card, press keys Selection Rule: If ATM is nearby → use ATM

GOMS assumes correct behavior. It does NOT model: confusion, learning, feedback, or errors.

Reference:

Dix et al., Human–Computer Interaction, 3rd ed., Pearson. Chapter 9, pp. 318–323 (GOMS Model)

3. Keystroke-Level Model (KLM)

KLM is a simplified, quantitative version of GOMS. It predicts how long a task will take for an expert user.

KLM Operators

  • K – Keystroke or tap
  • P – Pointing to a target
  • H – Homing (move hand between devices)
  • M – Mental preparation
  • R – System response time
  • B – Mouse button press

Example (ATM PIN entry): [M] think → [P] move finger → [K] press key

KLM assumes: expert users, no errors, and no need for feedback.

Reference:

Dix et al., Human–Computer Interaction, 3rd ed., Pearson. Chapter 9, pp. 323–327 (KLM Operators & Time Prediction)
