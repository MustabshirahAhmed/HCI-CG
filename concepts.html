1. Norman’s Action Cycle – Core Concepts

  • Focuses on how users think, act, and evaluate results.
  • Shows the gap between what users want and what the system allows.
  • Includes the idea of **Gulfs of Execution and Evaluation**.

Q1: What is Norman’s Action Cycle?

  • It explains how users move from goal → action → result → understanding.
  • It has 7 stages: Goal, Plan, Specify, Execute, Perceive, Interpret, Evaluate.

Q2: What is the Gulf of Execution?

  • It is the gap between what the user wants to do and what the system lets them do.
  • If buttons are unclear, the gulf becomes large.

Q3: What is the Gulf of Evaluation?

  • It is the gap between what the system shows and what the user understands.
  • If feedback is confusing, users don’t know what happened.

2. GOMS – Core Concepts

  • Used to describe how users perform tasks step by step.
  • Best for expert, error-free behavior.
  • Focuses on structure of tasks.

Q4: What does GOMS stand for?

  • G = Goals (what user wants)
  • O = Operators (actions user performs)
  • M = Methods (steps to reach the goal)
  • S = Selection Rules (how user chooses a method)

Q5: What is a Goal in GOMS?

  • It is the user’s main objective.
  • Example: Withdraw cash from ATM.

Q6: What is an Operator?

  • A basic action like clicking, typing, or pointing.

3. KLM – Core Concepts

  • Predicts how long a task will take.
  • Works for expert users with no mistakes.
  • Uses time values for each action.

Q7: What is KLM?

  • It stands for Keystroke-Level Model.
  • It measures time for low-level actions.

Q8: What are KLM Operators?

  • [K] – Key press
  • [P] – Point with mouse
  • [H] – Move hands
  • [M] – Mental preparation
  • [R] – System response time

Q9: What does KLM NOT model?

  • Thinking deeply
  • Understanding meaning
  • Making decisions

Summary – All Three Together

  • Norman → Focuses on thinking + feedback.
  • GOMS → Focuses on task structure.
  • KLM → Focuses on time prediction.
